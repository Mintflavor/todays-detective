## 🕵️‍♂️ 프로젝트: 시크릿 아카이브 (관리자 모드) 구현 계획 [완료]

### 1. 개요 (Overview)
게임 내 숨겨진 커맨드를 통해 접근 가능한 **[관리자 보안 구역]**을 구현합니다. 이 기능은 개발자 또는 관리자가 생성된 시나리오 데이터를 관리(삭제)하기 위한 목적으로 사용됩니다. UI/UX는 게임의 세계관을 해치지 않도록 '기밀 문서 보관소' 컨셉을 유지합니다.

**상태:** ✅ 구현 완료 (2025-12-10)

### 2. 백엔드 개발 (Backend)
**목표:** 시나리오 영구 삭제를 위한 API 엔드포인트 구축
*   **파일:** `backend/routes/scenarios.py`
*   **기능:** `DELETE /scenarios/{id}` 구현 완료.
*   **상태:** ✅ 완료

### 3. 프론트엔드 API (Frontend API)
**목표:** 백엔드와 통신하는 클라이언트 로직 추가
*   **파일:** `app/lib/api.ts`
*   **기능:** `deleteScenario` 함수 구현 완료.
*   **상태:** ✅ 완료

### 4. UI/UX 디자인 (Admin UI)
**목표:** 느와르 테마가 적용된 관리자 전용 인터페이스 구현
*   **컴포넌트:** `app/components/AdminScreen.tsx`
*   **디자인 컨셉: "The Burn Bag (소각장)"**
    *   **헤더:** "보안 구역 (SECURE AREA)" 표기 및 TOP SECRET 스탬프 적용.
    *   **언어:** 한국어 UI 적용 완료.
    *   **리스트 뷰:** 페이지네이션(10개 단위), 상세 정보 아코디언(개요, 피해자, 용의자, 진범) 구현 완료.
    *   **삭제 액션:** "기록 말소" 버튼 및 경고 모달 구현 완료.
*   **상태:** ✅ 완료

### 5. 히든 커맨드 로직 및 인증 (Secret Logic & Auth)
**목표:** 보안 커맨드 감지 및 2차 인증 시스템 구축
*   **훅(Hook):** `app/hooks/useSecretCommand.ts` (좌우 화살표 5회 반복 감지)
*   **인증:** `app/api/admin/verify/route.ts` 및 `AdminAuthModal.tsx` 구현 완료. `ADMIN_PASSWORD` 환경 변수 사용.
*   **상태:** ✅ 완료

### 6. 통합 (Integration)
**목표:** 게임 루프에 관리자 모드 및 인증 절차 탑재
*   **파일:** `app/page.tsx`
*   **흐름:** 인트로 화면에서 커맨드 입력 시 인증 모달 -> 비밀번호 입력 성공 시 관리자 화면 진입.
*   **상태:** ✅ 완료

### ⚠️ 주의사항
*   `NEXT_PUBLIC_API_URL` 환경 변수를 사용하여 백엔드 주소를 관리합니다. (기본값: `http://localhost:8000`)
